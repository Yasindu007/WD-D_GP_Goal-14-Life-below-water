Student 2, Content Developer

Home Page

Technical Description

The Home Page serves as the entry point to the Ocean Guardians website. I used a combination of HTML5 semantic tags like <nav>, <main>, and <footer> to structure the content logically. The navigation bar was styled with CSS Flexbox (display: flex) for responsive alignment, and a hamburger menu was implemented using JavaScript to toggle visibility on smaller screens. The background image uses a slow zoom animation (@keyframes slowZoom) to create a dynamic effect, enhancing visual appeal. Key CSS properties like position: absolute and z-index were used for layering the overlay and text over the image.

Accessibility

Accessibility was addressed by ensuring high contrast between text and background (white text on a dark overlay), using alt attributes for all images (e.g., alt="Ocean Guardians Logo"), and making the navigation keyboard-navigable with tabindex. The hamburger menu toggle is also accessible via keyboard events.

Table Page

Technical Description

The Table Page displays marine conservation efforts by country using an HTML <table> element. The structure includes <thead> for headers and <tbody> for data, styled with CSS to improve readability. Key properties include border-collapse: collapse for clean borders, text-align: left for content alignment, and :hover effects to highlight rows. Flags were added as images with class="flag" and styled with fixed dimensions (width: 30px) for consistency.

Accessibility

Accessibility was ensured by adding descriptive alt text to flag images (e.g., alt="Australia Flag") and using <th> elements with scope attributes (implied by context) to aid screen readers. The table’s contrasting colors (white background, blue headers) meet WCAG contrast guidelines.

Content Page (Content 02)

Technical Description

The Content 02 Page focuses on ocean conservation efforts, structured with <main> and multiple <div class="content-container"> sections. Each section pairs an image with text, styled using Flexbox for layout consistency. The button links (<a> tags) use id anchors for smooth navigation within the page. CSS animations (@keyframes fadeIn) enhance the overlay effect on the cover image, while border-radius and box-shadow add a polished look to containers.

Accessibility

Accessibility features include alt text for all images (e.g., alt="Content Image"), sufficient color contrast for text readability, and semantic HTML for screen reader compatibility. The back-to-top button (<a href="#top">) improves navigation for users with mobility issues.

Challenges and Lessons Learned

One challenge was ensuring the hamburger menu worked responsively across devices. I resolved this by debugging the JavaScript toggle function and adjusting media queries. Another issue was aligning table content consistently; I used fixed padding and image sizes to fix this. I learned the importance of testing across browsers and prioritizing modular CSS for maintainability.

Compliance

My pages adhere to web standards (HTML5, CSS3) and accessibility guidelines (WCAG 2.1), such as providing alt text and keyboard navigation. While I’m unaware of specific "Janet regulations," I ensured compliance with general best practices like valid markup, tested via W3C validators, and GDPR-friendly links in the footer (e.g., Privacy Policy).
