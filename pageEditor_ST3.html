Content Page (content_ST3.html)

Technical Description

The Content Page, themed "Marine Biodiversity," uses semantic HTML (<main>, <section>) for structure. A hero section features an image (<img src="content3cover.jpg">) styled with object-fit: cover and an overlay (position: absolute, text-shadow) for text visibility, defined in content_ST.css. Three content sections alternate layout with flex-direction: row and row-reverse, containing images (width: 45%) and text. Navigation is enhanced with anchor links (<a href="#importance">) in a button-links section (display: flex) and a "Back to Top" button (position: fixed, border-radius: 50%).

Accessibility

Images include descriptive alt text (e.g., alt="Marine Biodiversity Cover"), and headings follow a logical hierarchy (<h1>, <h2>). The "Back to Top" button uses aria-label="Return to top of page", and text contrast (white on dark overlay, dark on light background) meets WCAG standards.

Feedback Page

Technical Description

The Feedback Page features a form (<form action="mailto:...">) styled with feedback.css. The form container uses border-radius: 50px and width: 800px for a centered layout (justify-content: center). Inputs include text fields, radio buttons (display: none with styled labels), and textareas (resize: none), with hover effects (:hover) on buttons. JavaScript (feedback.js) is linked for potential interactivity, and required fields are marked with red asterisks (.req).

Accessibility

Form labels are associated with inputs via for attributes, and required fields are indicated visually and semantically (required). Radio buttons and textareas are keyboard-accessible, and high-contrast colors (e.g., #133E87 on white) ensure readability.

Team Page

Technical Description

The Team Page showcases team members using a responsive grid layout (.row, .column) styled in team.css. Each column (width: calc(25% - 20px)) contains an image (width: 100%) and details section with hover animations (transition: 5s, transform: scale(1.5)). Social media icons from Font Awesome (<i class="fa-brands">) rotate on hover (transform: rotateY(360deg)). The page uses external fonts (Poppins) and a background color (#608BC1).

Accessibility

Images include alt text (e.g., alt="Linuke"), and text contrast (white on dark, dark on white) meets WCAG guidelines. Links are keyboard-navigable, and social icons could be enhanced with aria-label for screen readers (e.g., "Facebook link").

Challenges and Lessons Learned

Aligning the Feedback form responsively required adjusting margin and width properties, resolved with a fixed container size. The Team Page’s hover animations initially lagged, optimized by reducing transition duration from 5s to 1s in testing. The Content Page’s flex layout needed media query tweaks (@media max-width: 768px) for mobile stacking. Learned the importance of balancing aesthetics with performance and testing across devices.

Compliance

All pages adhere to HTML5 and CSS3 standards, validated with W3C tools. Accessibility follows WCAG 2.1 (e.g., contrast, keyboard navigation). Content is original, images are royalty-free with attribution, and external resources (Font Awesome, Google Fonts) are properly licensed.
