User Profile Page

Technical Description

The User Profile Page uses HTML5 semantic elements (<main>, <div>) and is styled with profile.css. A cover video (<video>) is styled with object-fit: cover and runs via autoplay muted loop. The profile photo (<div>) uses border-radius: 50% and position: absolute. Sections like skills (<div class="skills-section">) feature progress bars (<div class="skill-progress">) with transition: width 0.5s, and goals (<div class="goal">) have hover overlays (transition: bottom 0.5s). Flexbox (display: flex) and a grid (grid-template-columns: repeat(6, 1fr)) structure the layout.

Accessibility

Images and icons include alt text (e.g., alt="Skill Icon"), form inputs use labels (<label for="name">), and the availability table uses <thead> and <tbody>. High-contrast colors (e.g., #133e87 on #cbdceb) and keyboard-navigable hover effects meet WCAG standards.

Sitemap Page

Technical Description

The Sitemap Page features an interactive SVG (<svg>) styled with Sitemap.css. Circles (<circle>) and ellipses (<ellipse>) represent pages, with hover effects (transition: fill 0.3s) shifting fill from #608bc1 to #133e87. Links (<a>) use data-image for tooltips, and connections are drawn with <line> elements (stroke: #cbdceb). Media queries (e.g., @media max-width: 1440px) ensure responsiveness.

Accessibility

SVG text labels (<text>) aid screen readers, and navigation icons have alt text. Contrast between #608bc1 circles and the #608BC1 background could be enhanced, but hover states improve visibility.

Content Page (content_ST4.html)

Technical Description

The Content Page, themed "Threats to Life Below Water," uses semantic HTML (<main>, <div>) and is styled with content_ST.css. A hero section features an image (<img src="Img/content4img/content4cover.jpg">) with object-fit: cover and an overlay (position: absolute, text-shadow). Three content sections (<div class="content-container">) alternate layout with flex-direction: row and row-reverse, containing images (width: 45%) and text. Navigation includes anchor links (<a href="#pollution">) in a button-links section (display: flex) and a "Back to Top" button (position: fixed, border-radius: 50%).

Accessibility

Images have descriptive alt text (e.g., alt="Content Image"), and headings follow a logical hierarchy (<h1>, <h2>). The "Back to Top" button could use aria-label="Return to top", though not explicitly coded. Text contrast (e.g., white on dark overlay) meets WCAG guidelines.

Challenges and Lessons Learned

The Profile Page’s video positioning required adjusting z-index and position: absolute for proper layering. The Sitemap’s SVG responsiveness needed overflow-x: scroll for smaller screens. The Content Page’s flex layout was refined with media queries (@media max-width: 768px) to stack content responsively. Learned to prioritize accessibility early, test animations for performance, and validate layouts across devices.

Compliance

All pages adhere to HTML5 and CSS3 standards, validated with W3C tools. Accessibility follows WCAG 2.1 (e.g., contrast, keyboard navigation). Content is original, images are royalty-free with attribution, and external resources are properly licensed.
